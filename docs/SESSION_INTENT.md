# セッション設計の意図

## Background

AirToss の相手認証は、近くの別人や中間者への誤送信を防ぐためにある。端末所有者自身からセッション情報を隠すことは目的ではない。モバイル OS のプロセス終了で長時間の転送を最初からやり直させるほうが、端末内保存より利用者への損失が大きい。

## Intent

- 認証確認は、見比べやすい 6 桁の数字に留める。人間に複雑な操作や長い文字列を要求しない
- 進行中の転送はプロセスより長く生存させる。再開状態の端末内保存を許容し、完了、拒否、中止後は残さない
- 通信相手の認証と経路上の暗号化は維持する。端末内攻撃への過剰な防御を理由に、再開性を落とさない
